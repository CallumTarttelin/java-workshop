OOP

Object-oriented programming is a programming paradigm based around objects. The objects represent things with fields and methods.

  • Object-oriented programming is a paradigm
  • Java is a popular example of a general purpose object oriented language
  • Java, unlike python, is not a pure OO language because not everything is an object

Classes, Fields, Methods, and Objects

Classes + Fields + Methods

Cat.java


                            public class Cat {
                            }

Cat.java


                            public class Cat {
                                public String name;
                                public boolean inside;
                            }

App.java


                            public class Cat {
                                public String name;
                                public int age;

                                public boolean isAdult() {
                                    return this.age > 1;

                                    this.name = name;

    Classes usually are named in PascalCase
    Classes by themselves aren't that useful
    Classes can hold fields to show properties of a class
    Classes can also have methods to show actions of a class
    These fields and methods are usually in camelCase
    These work as variables and functions associated with the class

  • Objects are instances of classes

Access Modifiers

You might have noticed a lot of things marked as public, classes and fields in Java can have limited access from other places

Access Modifiers

Example.java


                        class Example {
                            public Field publicField;
                            private Field privateField;
                            Field packagePrivateField;
                            protected Field protectedField;
                        }
  • A public class or field can be accessed anywhere
  • A private class or field can only be accessed in the enclosing class
  • A package private field can only be accessed by the same package (folder)
  • A protected field can only be accessed by the enclosing class or its children
Most often you will see public and private, where public is to be accessed from anywhere and private is not supposed to be changed.

Constructors

Constructors

Example.java


                        class Example {
                            private Field field;

                            public Example(Field field) {
                                this.field = field;
                            }

                            private Example(FieldBox fieldBox) {
                                this.field = fieldBox.unbox();
                            }

                            Example() {
                            }
                        }
Whilst classes with no constructors defined have a default constructor which gives all fields default values (usually null unless primitive), you can also define your own.
  • They can take values to give directly to the fields
  • You can give them other things to compute the value for starting fields
  • You don't have to initialize anything, by default fields will be given value of null, unless primitive
Constructors can also have different access modifiers, these work the same as with methods and can allow for classes that cannot be constructed. This may be useful if they only have static fields and methods.

OOP concepts

Encapsulation

Uses methods to prevent misuse of fields

Encapsulation

Person.java


                        public class Person {
                            private String name;

                            public String getName() {
                                return this.name;
                            }

                            public void setName(String name) {
                                this.name = name;
                            }
                        }
This is usually achieved by using methods to get and set data correctly, this allows for defining public use making refactoring easy in case fields change in the future, you can be sure they aren't used. In java these are commonly getters and setters like these, these do get quite out of control in larger classes and some tools can be used to generate these for you.

Inheritance

The idea of classes extending the functionality of other classes

Inheritance

Person.java


                        public class Person {
                            protected int age;
                            public String name;

                            public Person(int age, String name) {
                                this.age = age;
                                this.name = name;
                            }

                            public String favouriteColor() {
                                return "#564374";
                            }
                        }
This is just a generic class, there should be nothing new here

Inheritance

User.java


                        public class User extends Person {
                            private Long id;

                            public User(int age, String name, Long id) {
                                super(age, name);
                                this.id = id;
                            }

                            public int getAge() {
                                return this.age;
                            }
                        }
Here we have used extends to inherit the parent class. This allows us to add fields and provide new constructors, the super() keyword allows us to access parent constructor getAge accesses a protected field, it can do that as it is a child of person

Inheritance

App.java


                        public class User extends Person {
                            private Long id;

                            public User(int age, String name, Long id) {
                                super(age, name);
                                this.id = id;
                            }

                            public int getAge() {
                                return this.age;
                            }

                            @Override
                            public String favouriteColor() {
                                return "#109f40";
                            }
                        }
Here we can also add a method that overrides the parent method, the @Override is technically optional but stops compiler warnings

Interfaces

Interfaces are used to describe what features a class should have. It does not define any of the behaviour itself

Interfaces

Shape.java


                        public interface Shape {
                            Double getArea();

                            default Integer getAreaInt() {
                                return getArea().intValue();
                            }
                        }
Interfaces are defined similarly to classes, all the methods have no implementation unless default is set. All methods are public by default.

Interfaces

Square.java


                        public class Square implements Shape {
                            private final Double x;
                            private final Double y;

                            public Square(Double x, Double y) {
                                this.x = x;
                                this.y = y;
                            }

                            @Override
                            public Double getArea() {
                                return x * y;
                            }
                        }
Interfaces are implemented with the implements keyword Methods from interface are marked with @Override and implemented, the default method will be implemented automatically

Interfaces

App.java


                        Shape shape = new Square(3.0, 4.0);
                        shape.getArea();
                        // 12.0

App.java


                        public Double area(Shape shape) {
                            return shape.getArea();
                        }
You can specify the interface as the type accepted and accept any class that implements the interface. This allows for different methods to be called within a method dependent on class.

Polymorphism

Is a name used when calling code that is agnostic to the class it is operating on. In the interface example, if there was a Circle as well as a Square the area method would be polymorphic.
